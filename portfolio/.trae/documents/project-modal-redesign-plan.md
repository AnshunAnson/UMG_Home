# 项目弹窗卡片重新设计计划

## 目标
使用前端skill和多个subagent重新设计Projects.tsx中的项目弹窗卡片(ProjectModal)，提升UI/UX质量和视觉体验。

## 当前问题分析

### 1. 布局问题
- 左右分栏布局(38%/62%)在宽屏下右侧图片区域过大
- 左侧信息区域文字密度过高，缺乏视觉层次
- 图片展示区域排版单调，缺乏交互感

### 2. 视觉层次问题
- 标题、描述、列表项之间的视觉区分度不够
- 技术栈标签样式过于简单
- 缺乏项目主题色的深度应用

### 3. 交互体验问题
- 图片hover效果单一
- 缺少图片标题/说明的展示
- 关闭按钮位置不够醒目

## 设计方向

### 整体风格
- **Glassmorphism(毛玻璃)** + **Neon(霓虹)** 风格
- 深色主题配合项目主题色点缀
- 流畅的动画过渡效果

### 布局重构
- 采用**Bento Grid**布局替代传统左右分栏
- 信息卡片化，提升可读性
- 图片区域采用**Masonry**或**Grid**布局

## 实施步骤

### Phase 1: 设计系统生成
**Agent**: ui-designer (设计专家)
**任务**:
1. 生成项目弹窗的设计系统
2. 定义颜色、间距、动画规范
3. 提供组件结构建议

### Phase 2: 布局架构设计
**Agent**: frontend-architect (前端架构师)
**任务**:
1. 设计新的弹窗布局结构
2. 规划组件拆分
3. 定义动画实现方案

### Phase 3: 代码实现
**Agent**: frontend-architect (前端架构师)
**任务**:
1. 重构ProjectModal组件
2. 实现新的布局系统
3. 添加动画效果

### Phase 4: 质量验证
**Agent**: verification-agent (验证专家)
**任务**:
1. 验证UI/UX规范符合性
2. 检查响应式适配
3. 确认动画性能

## 技术栈
- React + TypeScript
- Tailwind CSS
- Framer Motion
- Lucide Icons

## 设计原则
1. **一致性**: 保持与现有项目卡片风格一致
2. **可读性**: 信息层次清晰，易于阅读
3. **交互性**: 丰富的hover和点击反馈
4. **响应式**: 适配各种屏幕尺寸
5. **性能**: 动画流畅，不卡顿
